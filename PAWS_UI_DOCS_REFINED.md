# Руководство по обновлению документации Paws UI (NPM Package)

Привет! Я проанализировал текущую версию документации. Она стала намного лучше, но ее нужно очистить от лишних деталей и уточнить технические моменты использования библиотеки.

Вот список правок, которые нужно внести. **Важно:** забудь про контекст "фреймворка Paws" или конкретных плагинов, пиши документацию для пользователя NPM-пакета, который хочет использовать этот UI-кит.

### 1. Темизация и CSS Layers (Критично)

В нашей системе темы работают через CSS Cascade Layers (`@layer`). Базовые стили и цвета подгружаются через внешние CSS-файлы хост-приложения.

- **Инструкция:** Удали упоминание о том, что нужно обязательно вешать класс `paws--dark-theme` на корневой элемент. Библиотека должна полагаться на то, что CSS-переменные (например, `--paws-color-bg-darkest`) уже объявлены глобально.
- **Что добавить:** Упомяни, что в `index.html` пользователя должны присутствовать ссылки на базовые стили тем (информацию об этом мы дадим в общем гайде по разработке плагинов, здесь достаточно сказать, что кит использует слои `@layer base, custom`).

### 2. УдалениеStorybook из документации

- **Инструкция:** Ссылок на Storybook в документации для разработчика (пользователя пакета) быть **не должно**. Сторибук используется только внутри команды при разработке самой библиотеки. Вместо этого в `components.md` дай больше текстовых примеров кода.

### 3. Синхронизация состояния (Config Sync)

- **Уточнение:** Метод `setPawsUiConfig` нужен не для сохранения настроек в базу данных, а только для управления **поведением** UI-кита "здесь и сейчас".
- **Инструкция:** Опиши, что через этот конфиг пользователь может:
  1. Включать/выключать подсказки (`showTooltips`).
  2. Указывать текущую базу темы (`theme: 'dark' | 'light'`) для корректной работы компонентов, которые отрисовываются через Portals (например, Tooltips), если им нужно знать контекст темы для JS-логики.

### 4. Шрифты и Анимации

- **Инструкция:** Упомяни, что за плавные переходы цветов отвечает файл `paws-theme-base.css` (который обычно подключен в хост-приложении). Компоненты библиотеки должны корректно анимироваться, используя класс `.paws-theme-transitioning` на уровне `body`.
- **Шрифты:** Подтверди, что шрифт "Wix Madefor Display" является стандартом, и библиотека ожидает его наличия (или подтягивает сама).

### 5. Структура `getting-started.md`

- **Инструкция:** Упрости пример `App.vue`. Теперь он должен выглядеть так:
  ```html
  <template>
  	<main class="plugin-wrapper">
  		<PawsButton tooltip="Click me!">Button Content</PawsButton>
  		<!-- Контейнер для подсказок, обязательно один на приложение -->
  		<PawsTooltip />
  	</main>
  </template>
  ```
  (Никаких принудительных классов тем в шаблоне).

### 6. Компоненты и сетка

- **Инструкция:** В `components.md` сделай акцент на том, что все интерактивные элементы имеют **фиксированную высоту** (например, 24px для инпутов, 32px для кнопок меню). Это ключевое правило для сохранения консистентности интерфейса.

**Итог:** Убери всё лишнее, сосредоточься на API пропсов и правильной инициализации пакета. Документация должна быть максимально "сухой" и технической.
